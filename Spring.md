# Spring의 4대 요소 (IoC, DI, PSA, AOP)

## IoC
Inversion of Control (제어의 역전)은 객체의 생성부터 생명주기 관리까지 모든 제어권을 개발자가 아닌 프레임워크인 Spring 컨테이너가 가져간다는 원칙
### 장점
1. 객체 관리 부담 해소: 개발자는 비즈니스 로직에만 집중하고, 객체를 언제 만들고 언제 없앨지는 Spring이 알아서 처리
2. 유연한 설계: 객체 간의 결합도가 낮아져 설정만 변경하면 앱의 동작을 쉽게 바꿀 수 있음
### 만약 위반한다면?
개발자가 모든 객체를 직접 new 키워드로 생성하고 관리해야 하므로, 프로젝트 규모가 커지면 커질수록 객체 간의 관계가 꼬이고 유지보수가 불가능해짐

## DI
Dependency Injection (의존성 주입)은 SOLID에서의 DIP를 실제로 구현하는 방법 중 하나로, 객체가 필요한 의존 객체를 직접 찾지 않고 외부에서 주입해주는 방식
### 장점
1. 테스트 용이성: 실제 DB 연결 대신 Mock(가짜 객체)를 주입하여 단위 테스트를 진행하기 때문에 매우 편리함
2. 재사용성 향상: 인터페이스를 기반으로 주입받으면, 구현체가 바뀌어도 사용하는 클래스의 코드는 수정할 필요가 없음
### 만약 위반한다면?
클래스 내부에서 특정 구현 클래스를 집적 의존하게 되면서 부품 하나를 갈아끼우기 위해 코드 전체를 다시 짜야하는 상황이 발생함

## AOP
Aspect Oriented Programming (관점 지향 프로그래밍)은 보안, 프랜잭션 등 여러 비즈니스 로직에 공통적으로 등장하는 기능을 분리해서 별도의 모듈로 관리하는 기술
### 장점
1. 코드 중복 제거: 모든 메서드에 일일히 넣어야 했던 시간 측정 코드를 한 곳에서 관리할 수 있음
2. 핵심 로직 집중: 비즈니스 로직 코드 안에 부가적인 기능이 섞이지 않아 가독성이 상승함
### 만약 위반한다면?
메서드에 트랜잭션 설정이나 로그 출력 코드를 복사/붙여넣기를 해야 하며, 변경 사항이 생길 때마다 그 메서드들을 일일히 찾아다니며 수정해야 함

## PSA
Portable Service Abstraction (일관된 서비스 추상화)은 환경이나 기술의 변화와 상관 없이 일관된 방식으로 서비스에 접근할 수 있게 해주는 추상화 계층
### 장점
1. 기술 독립성: DB 접근 기술을 MyBatis에서 JPA로 바꾸거나 톰캣 서버를 네티로 바꾸더라도 기존 비즈니스 로직 코드는 그대로 유지됨
2. 학습 비용 감소: 특정 기술의 세부 구현을 몰라도 추상화된 인터페이스 사용법만 알면 기능 구현 가능
### 만약 위반한다면?
사용하는 외부 라이브러리나 서버 환경이 바뀔 때마다 그 기술에 종속된 코드를 전부 다시 작성해야 함.
