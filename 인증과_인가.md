## 인증(Authentication)
사용자가 자기 자신이 누구임을 주장할 때, 서버가 이것을 확인하는 과정
### 주요 수단:
- 지식 기반: 아이디/패스워드
- 소유 기반: OTP, 스마트폰 인증, 보안 키
- 속성 기반: 지문, 홍채, 얼굴 인식
### Spring Security에서의 역할:
- 사용자가 입력한 정보와 DB의 정보를 비교해서 Authentication 객체를 생성함
- 이 객체가 생성이 되면 로그인이 된 상태

## 인가(Authorization)
인증된 사용자가 특정 리소스에 접근할 권한이 있는지 결정하는 과정
### 주요 방식:
- RBAC(Role-Based Access Control): 역할 기반
- ABAC(Attribute-Based Access Control): 속성 기반
### Spring Security에서의 역할:
- hasRole('ADMIN')과 같은 설정을 통해 특정 API 호출을 허용하거나 막음
- 인증을 통과했지만 권한이 없으면 403: Forbidden 에러가 발생함
