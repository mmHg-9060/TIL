# Spring의 핵심 어노테이션 (Annotations)

## @Component
@Component는 개발자가 직접 작성한 클래스를 Spring 컨테이너가 생성하고 관리하는 빈(Bean)으로 등록하라고 지시하는 어노테이션
### 장점
1. 자동 스캔 및 등록: XML이나 설정 파일에 일일이 객체를 등록할 필요 없이, 컴포넌트 스캔을 통해 스프링이 알아서 클래스를 찾아서 객체로 생성해줌
2. 역할의 명확화: 하위 어노테이션을 사용하면 해당 클래스가 비즈니스 로직을 처리하는지 DB에 접근하는지 코드를 여는 순간 직관적으로 파악이 가능함
### 만약 사용하지 않는다면?
수백~수천개의 클래스를 설정 파일에 직접 new 키워드를 사용해 수동으로 등록해야 하므로 설정 코드가 비대해지고, 누락 실수가 잦아짐

## @Autowired
@Autowired는 Spring 컨테이너가 만들어둔 빈을 객체가 필요한 곳에 자동으로 연결(의존성 주입)해주는 어노테이션
### 장점
1. 의존성 해결 자동화: 객체가 자신이 사용할 다른 객체를 직접 찾거나 생성할 필요 없이, 스프링이 타입에 맞는 개체를 찾아줌
2. 보일러플레이트 코드 감소: 의존성을 주입받기 위한 보갑한 팩토리 패턴이나 객체 검색 로직을 작성할 필요가 없이 코드가 깔끔해짐
### 만약 사용하지 않는다면?
클래스 내부에서 필요한 객체를 직접 생성해야 하므로 객체 간의 결합도가 극도로 높아지며, 단위 테스트를 할 때 가짜 객체(Mock)를 끼워넣기가 거의 불가능해짐

## @RestController
@RestController는 해당 클래스가 RESTful 웹 서비스의 요청을 처리하는 곳임을 명시하며, 반환값을 뷰가 아닌 json이나 XML같은 순수 데이터 형태로 바로 응답하게 해주는 어노테이션 (Controller + ResponseBody)
### 장점
1. 직관적인 API 개발: 각 메서드마다 데이터 반환용 어노테이션을 붙일 피룡 없이, 자바 객체를 반환하기만 하면 스프링이 알아서 json 포맷으로 변환해 클라이언트에게 전달함
2. 프론트엔드와 백엔드의 분리: 화면 렌더링 없이 데이터만 주고받는 구조이므로 React, Vue와 같은 SPA(Single Page Application)이나 모바일 앱과 통신하는 서버를 만들 때 매우 적합함
### 만약 사용하지 않는다면?
데이터만 반환해야 하는데 스플링이 계속 HTML을 찾으려고 시도하다가 404 에러를 반환하거나 모든 반환값에 json 변환 처리 코드를 수동으로 붙여야 함.

## @Transcational
@Transactional은 메서드나 클래스에 데이터베이스 트랜잭션을 적용하여, 내부의 여러 작업이 모두 성공하면 Commit(반영)하고 하나라도 실패하면 Rollback(처음 상태로 회귀)하는 어노테이션
### 장점
1. 데이터 정합성 보장: 송금 로직 처리 중 서버가 다운되건 ㅏ에러가 발생해도, 돈이 중간에 증발하거나 복사되지 않고 안전하게 원래 상태로 복구됨
### 만약 사용하지 않는다면?
복잡한 비즈니스 로직 중간에 예외가 터졌을 때 앞서 실행된 쿼리들만 DB에 덩그러니 반영되어 심각한 데이터 불일치(돈은 나갔지만 결제는 안 되는 상황)이 발생함.
